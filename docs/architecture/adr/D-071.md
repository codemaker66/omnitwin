# D-071 — Mermaid as on-demand visualization

Status: proposed
Date: 2026-04-25
Authors: Blake
Supersedes: none

## Context

Founder reports aesthetic quality affects motivation. Diagrams need to feel premium, not throwaway. Mermaid renders natively in GitHub, VS Code, and Claude. Excalidraw is the escape hatch for once-a-quarter "let me think visually" sessions.

## Decision

Five recurring diagrams in `docs/diagrams/` as Mermaid markdown files. Aesthetic theme committed in `docs/diagrams/_theme.md`: Aman/Four Seasons-adjacent palette, sentence case in node labels, rounded brackets for soft corners, max 12 nodes per diagram. Aesthetic quality is treated as a first-class engineering concern. Defer auto-generation (e.g., a script that builds diagrams from ADR front-matter) until the manual flow has been used for two weeks and proven useful.

## Consequences

Every diagram in the repo follows the theme; updating the theme updates all diagrams. Excalidraw VS Code extension handles freehand cases. D2 deferred indefinitely.

## Alternatives considered

- **Excalidraw as primary** — rejected. Worse for AI-assisted generation than Mermaid.
- **D2** — rejected. Premature complexity for solo-founder scale.
- **Lucidchart / draw.io** — rejected. SaaS, violates D-070.
- **No visualization** — rejected. Loses the founder-motivation benefit and makes architecture harder to discuss with non-technical reviewers.
