# D-010 — Pose-frame indirection

Status: proposed
Date: 2026-04-25
Authors: Blake
Supersedes: none

## Context

The current pipeline assumes COLMAP is the only pose-estimation tool. COLMAP is excellent at full-resolution structure-from-motion when the venue has dense, well-distributed coverage, but it struggles when capture is sparse, incremental, or constrained to hard-to-reach regions (Trades Hall balcony, organ alcove). Recent learned methods — MapAnything (Apple), VGGT (Microsoft Research) — produce competitive poses from far fewer views and are practical for follow-up captures that COLMAP would reject. They have a hard limit: input resolution caps at 512–518 px, which is below the resolution we want for full-venue capture and reconstruction.

## Decision

Build a `PoseFrame` interface (D-009 family) so COLMAP, MapAnything, and VGGT can swap per-venue or per-capture. Defaults:

- **Default:** COLMAP (full-resolution SfM, dense initial captures).
- **Fallback:** MapAnything for sparse-view incremental capture (balconies, alcoves, follow-up scans).
- **Optional:** VGGT for ablation comparison and as a research path.

Each pose backend produces the same interface contract: per-capture extrinsics in the canonical venue coordinate frame, plus reprojection-error metadata (consumed by D-011 spatial confidence budget).

## Consequences

Hard-to-reach capture regions stop being reconstruction blockers — sparse-view methods cover them. Pose backend selection becomes a per-venue decision recorded in the AssetVersion bundle (D-014), not a hard-coded pipeline assumption. Existing Trades Hall captures continue with COLMAP and don't need re-registration. A `PoseFrame` interface unblocks the broader VSIR-0 abstraction (D-009).

Falsifying experiment: regenerate the Trades Hall pose graph with MapAnything; accept the swap if E57 alignment is ≤5 mm RMS, the same threshold COLMAP must clear.

## Alternatives considered

- **Keep COLMAP-only** — rejected. Forecloses on incremental capture of hard-to-reach venue regions and on the broader VSIR-0 indirection.
- **Switch wholly to MapAnything or VGGT** — rejected. Resolution cap (512–518 px) means they cannot replace full-resolution COLMAP for primary venue captures.
- **Hybrid: run all three on every venue** — rejected. Triple compute cost; no clear quality gain over the default+fallback policy.
