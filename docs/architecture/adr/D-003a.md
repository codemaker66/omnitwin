# D-003a — Layered venue radiance graph

Status: proposed
Date: 2026-04-25
Authors: Blake
Supersedes: D-003

## Context

The five-asset Genjutsu pipeline (D-003) is a strong v1 recipe but freezes implementation as architecture. Synthesis of four deep research reviews (two ChatGPT, two Claude) converged on the verdict that the durable abstraction is a typed spatial-layer graph with interchangeable render recipes — the five assets are one valuable instantiation of that, not the ontology of the business. Standards are moving fast (Khronos KHR_gaussian_splatting RC Feb 2026, OpenUSD v26.03 added `UsdVolParticleField3DGaussianSplat`).

## Decision

Reframe the pipeline as a typed spatial-layer graph (VSIR-0). The graph has typed nodes for CaptureSources, GeometricSubstrate, PoseFrame, BaseAppearance, ResidualAppearance, DeliveryFormat. The current five assets (panoramas, OBJ, cropped splat, E57, COLMAP poses) populate these node types in v1. Future implementations (BakedSDF base, MapAnything poses, neural residuals, glTF/USD delivery) plug in without rewriting the pipeline. See D-009 for the formal VSIR-0 specification.

## Consequences

Schema work in code reflects typed nodes, not asset filenames. Switching cost between renderer recipes drops from "rewrite pipeline" to "swap a node." Future ADRs about specific implementations don't need to amend D-003a — they just specify which nodes they populate.

## Alternatives considered

- **Keeping D-003 unchanged** — rejected. Locks 2026 implementation as company doctrine; fails when standards or backends shift.
- **Full LLVM-style optimizer with multiple passes** — rejected. Premature for solo founder; defer to v2 per Claude-DR Part 2.
- **A bespoke "VSP" published standard** — rejected for now. Defer until 50+ certified venues per Claude-DR §10.
