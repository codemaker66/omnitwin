# D-012 — Provenance and truth-mode separation

Status: proposed
Date: 2026-04-25
Authors: Blake
Supersedes: none

## Context

Generative reconstruction tools (CAT3D, ReconFusion, NVIDIA Cosmos, Marble) can fill in plausible geometry from sparse capture. They are useful for visualization — a planner showing a client what the room *could* look like — but the geometry they produce is not measured. It is plausible-looking inference. If that inferred geometry is silently mixed with measured geometry and a planner uses it to confirm capacity or compliance, the system has misled them. The harder this distinction is to ignore at the architectural level, the safer the product.

## Decision

Every scene graph element carries an explicit `provenance` field:

- **measured** — derived from raw capture (LiDAR, photogrammetry, calibrated stills).
- **inferred** — derived from measured data via a deterministic, audit-able transform (mesh decimation, ICP alignment).
- **generated** — produced by a generative model (CAT3D, ReconFusion, Marble, Cosmos, generative inpainting).

Runtime exposes three modes:

- **truth** — only `measured` and `inferred` elements are visible.
- **imagination** — all three provenance levels visible; `generated` regions get a visible badge.
- **hybrid** — same as imagination, but with the ability to toggle generated content per region.

Operational handoffs — BEO export, vendor pack, capacity certificates, fire-code documents, hallkeeper PDFs — are produced from the truth-mode scene only. The export pipeline asserts no `generated` element is present and refuses to produce the document otherwise. Generative content (CAT3D, ReconFusion, Marble, Cosmos) lives in imagination mode only.

## Consequences

A hard rule emerges: a planner cannot use imagined geometry as real measurement. The mode toggle is visible in the UI so the planner knows which mode they are in. Operational artifact generators (PDF render, vendor pack) get a small extra check (assert all elements are non-generated) that is cheap to implement and load-bearing for legal/compliance use. Generative reconstruction work gets unblocked because there is now a safe place for it to live.

## Alternatives considered

- **No provenance metadata** — rejected. Generative geometry would silently mix with measured, exactly the failure mode this ADR exists to prevent.
- **Provenance metadata but no runtime mode** — rejected. Without truth mode, the only enforcement is documentation, which planners under deadline pressure will skip.
- **Three separate scenes per provenance level** — rejected. Triples the scene-graph complexity and makes mode transitions expensive instead of a flag toggle.
