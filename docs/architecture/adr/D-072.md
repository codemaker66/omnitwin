# D-072 — task-master-ai not adopted (markdown task table suffices)

Status: proposed
Date: 2026-04-25
Authors: Blake
Supersedes: none

## Context

A previous Claude session recommended task-master-ai (npm package, MCP-integrated, PRD-driven workflow). On review, the overhead doesn't pay back at solo-founder scale: T-NNN naming conflicts with the package's own task IDs; PRD-driven workflow doesn't match the founder's actual flow (synthesize → decide → execute, not PRD-write-parse-execute); a markdown task table in `docs/state/tasks.md` provides the same human-and-AI-readable surface without tool overhead.

## Decision

Use a markdown task table in `docs/state/tasks.md` as the task tracking layer. Defer task-master-ai adoption. Reopen this decision if multi-agent task coordination becomes a real need (e.g., hires, parallel agent workstreams).

## Consequences

Task tracking is one file in git: version-controlled, diffable, mergeable. Trade-off: no built-in dependency graph generation, no automated complexity scoring, no agent orchestration. These are acceptable losses at current scale.

## Alternatives considered

- **task-master-ai** — deferred. Reopen when multi-agent need is real (hires, parallel agent workstreams).
- **GitHub Projects** — rejected. Creates a parallel source of truth.
- **No task tracking** — rejected. Loses the ability to see dependencies and prioritize work coherently.
