# D-004a — Projective texturing as v1 base appearance

Status: proposed
Date: 2026-04-25
Authors: Blake
Supersedes: D-004

## Context

D-004 commits projective texturing as the base renderer. Deep research synthesis confirms it remains the right v1 default for matte architectural interiors with dense panoramic coverage but flagged that calling it "the base renderer" rather than "v1 base implementation" forecloses on a `BaseAppearance` interface that future renderers (neural texture, BakedSDF-style mesh + view-dependent appearance, learned blend) would need.

## Decision

Keep projective texturing as the v1 default behind a `BaseAppearance` interface (D-009 provides the interface). Treat it as an implementation, not a forever-form commitment. Falsifying experiment: on the Trades Hall plaster moldings, compare projective texture base, BakedSDF-style mesh + spherical-Gaussian appearance, and cropped splat over OBJ. If an alternative wins by ≥0.02 LPIPS at ≥30 FPS on M2, schedule replacement.

## Consequences

Layer 1 work continues as-is in v1. New abstraction work (the interface) is small (~1 week). The bake-off (D-009 dependency on D-004a) gives an empirical answer about whether replacement is worth the engineering cost. Nothing in the active rendering path changes today; this is structural, not behavioural.

## Alternatives considered

- **Hardening D-004 as forever-form** — rejected. Prevents BakedSDF/neural exploration.
- **Replacing projective texturing now** — rejected. Premature without bake-off evidence.
- **Deferring all base-renderer work until standards land** — rejected. Loses the v1 shipping path.
