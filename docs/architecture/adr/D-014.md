# D-014 — Venue Artifact Factory

Status: accepted
Date: 2026-04-26
Authors: Blake
Supersedes: none

## Context

Today's pipeline blurs training and runtime. CUDA versions, Python toolchains, and trainer-specific quirks leak into web work; runtime engineers debug fused-ssim wheel mismatches that have nothing to do with rendering. The deep research synthesis flagged that the durable abstraction is a *boundary*: training is a process that produces a deterministic artifact bundle; runtime consumes the bundle. Each side can evolve without the other being affected. Local Windows splat training has already been deprecated (D-006a) in favor of RunPod A100 80GB; the rest of the boundary needs the same formalization.

## Decision

Stand up a **Venue Artifact Factory**: a formal boundary between training and runtime.

- **Input:** raw venue capture — E57 LiDAR, panoramas, fisheye stills, OBJ mesh, raw camera poses, semantic mask source images.
- **Process:** the factory runs the appropriate trainers and processors (gsplat MCMC + bilateral grid for pinhole captures, 3DGUT for distorted captures per D-006a; COLMAP, MapAnything, VGGT for poses per D-010; semantic segmentation; depth pack pre-computation for D-004 projection occlusion testing). RunPod A100 80GB is the canonical training environment.
- **Output:** a signed `AssetVersion` bundle containing:
  - canonical transforms (venue coordinate frame, per-capture extrinsics)
  - geometric substrate (decimated mesh, LOD manifests)
  - pano atlas + depth packs
  - splat / residual layers (cropped + archival)
  - semantic masks (reflective, transparent, planar, etc.)
  - QA reports (alignment RMS, ICP residuals, COLMAP reprojection error, eval PSNR/LPIPS)
  - calibration residuals
  - render recipes (which VSIR-0 family implementation each layer uses)
  - cryptographic signature
- **Runtime consumption:** runtime imports only the bundle. No Python, no CUDA versions, no trainer awareness. The artifact contract is the only interface.

The bundle schema is tracked in `state/asset_versions.json` (one entry per signed bundle) and lives long-term in R2.

## Consequences

Web/runtime engineers stop debugging trainer environments. Trainer engineers stop hand-walking new artifacts into runtime tests. New trainers (a future BakedSDF base, a learned blender) plug into the factory by emitting a compliant bundle. The bundle signature plus the QA reports give us provenance auditability — D-012 (truth-mode) checks signatures at handoff time.

Implementation specifics defined in D-016 (RunPod-canonical training environment) and the operator runbook at `infra/runpod/RUNBOOK.md`. The bundle format and trust boundary are now sufficiently concrete to drive T-001 through T-005 implementation.

## Implementation contract

### Bundle format (v1)

The factory produces a tarball whose extracted root contains exactly the following files. Names are fixed; presence of every file except `bilateral_grid.bin` is mandatory.

| Filename | Purpose |
|---|---|
| `scene.ply` | gsplat-native canonical model artifact (the trained Gaussians). |
| `training_config.json` | Full config snapshot — every flag passed to the trainer, every override applied. |
| `training_metrics.jsonl` | Per-iter loss, PSNR, SSIM, LPIPS as it ran. One JSON object per line. |
| `eval_holdout.json` | Held-out view metrics (PSNR, SSIM, LPIPS-Alex per image plus summary means) and WebGL FPS measurements where available. |
| `hardware.json` | GPU model, CUDA runtime version, driver version, RunPod pod ID, RunPod region. |
| `git_state.json` | Trainer code commit SHA, branch, dirty flag, remote URL — captured at run time. |
| `colmap_input.json` | COLMAP scene metadata: camera count, image count, point cloud bbox, intrinsics summary. |
| `bilateral_grid.bin` | OPTIONAL — included only when bilateral grid post-processing was enabled. Raw float32 grid contents. |
| `manifest.json` | Top-level descriptor: schema version, venue ID, run ID, signature placeholder, and SHA-256 of every file above. |

The manifest is the canonical entry point. SHA-256 of every other file in the bundle is recorded in `manifest.json.files[].sha256`.

### Signing posture (v1)

`manifest.json.signature` is present in v1 with a placeholder shape:

```json
"signature": {
  "status": "placeholder",
  "algorithm": null,
  "key_id": null,
  "value": null
}
```

Per D-013 future work, v1 ships with this placeholder. Production signing — Ed25519 in KMS, with Sigstore as the v2 migration target — is implemented when the AssetVersion table lands (T-018) and backend ingestion is built (T-053). The placeholder is intentionally non-empty so that the schema doesn't change between v1 and v2; only the `status`, `algorithm`, `key_id`, and `value` fields acquire real values.

### Trust boundary

Training pods write **unsigned candidate bundles** to a known R2 prefix:

```
venviewer-training-outputs/{venue_id}/{run_id}/
```

Pods carry R2-scoped credentials only — write to the outputs bucket, read from the inputs bucket. They do NOT carry Fastify credentials, database credentials, or signing keys. Backend ingestion (T-053) is the trust gate:

1. Pull the candidate bundle from R2.
2. Re-compute SHA-256 of every file and verify against `manifest.json.files[].sha256`.
3. Verify the bundle's structural shape matches the v1 contract above.
4. Sign the manifest with the production key (placeholder fields acquire real values).
5. Promote the bundle into the `AssetVersion` table with the signed manifest.

Until ingestion succeeds, the bundle is treated as untrusted. A failed integrity check at any step rejects the bundle and surfaces the failure to the operator.

The full bundle format spec, including JSON schemas for each file, lives at `docs/specs/runpod-training-contract.md`.

## Alternatives considered

- **Keep training and runtime intertwined** — rejected. Continues the current debt: every Spark/Three.js work session bumps into Python toolchain issues.
- **Bundle schema as documentation only, no signing** — rejected. Without the signature, D-012's truth-mode assertion has nothing to anchor to.
- **Per-trainer bundles with no shared schema** — rejected. Defeats the boundary; runtime ends up trainer-aware again.
