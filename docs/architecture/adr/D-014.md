# D-014 — Venue Artifact Factory

Status: proposed
Date: 2026-04-25
Authors: Blake
Supersedes: none

## Context

Today's pipeline blurs training and runtime. CUDA versions, Python toolchains, and trainer-specific quirks leak into web work; runtime engineers debug fused-ssim wheel mismatches that have nothing to do with rendering. The deep research synthesis flagged that the durable abstraction is a *boundary*: training is a process that produces a deterministic artifact bundle; runtime consumes the bundle. Each side can evolve without the other being affected. Local Windows splat training has already been deprecated (D-006a) in favor of RunPod A100 80GB; the rest of the boundary needs the same formalization.

## Decision

Stand up a **Venue Artifact Factory**: a formal boundary between training and runtime.

- **Input:** raw venue capture — E57 LiDAR, panoramas, fisheye stills, OBJ mesh, raw camera poses, semantic mask source images.
- **Process:** the factory runs the appropriate trainers and processors (gsplat MCMC + bilateral grid for pinhole captures, 3DGUT for distorted captures per D-006a; COLMAP, MapAnything, VGGT for poses per D-010; semantic segmentation; depth pack pre-computation for D-004 projection occlusion testing). RunPod A100 80GB is the canonical training environment.
- **Output:** a signed `AssetVersion` bundle containing:
  - canonical transforms (venue coordinate frame, per-capture extrinsics)
  - geometric substrate (decimated mesh, LOD manifests)
  - pano atlas + depth packs
  - splat / residual layers (cropped + archival)
  - semantic masks (reflective, transparent, planar, etc.)
  - QA reports (alignment RMS, ICP residuals, COLMAP reprojection error, eval PSNR/LPIPS)
  - calibration residuals
  - render recipes (which VSIR-0 family implementation each layer uses)
  - cryptographic signature
- **Runtime consumption:** runtime imports only the bundle. No Python, no CUDA versions, no trainer awareness. The artifact contract is the only interface.

The bundle schema is tracked in `state/asset_versions.json` (one entry per signed bundle) and lives long-term in R2.

## Consequences

Web/runtime engineers stop debugging trainer environments. Trainer engineers stop hand-walking new artifacts into runtime tests. New trainers (a future BakedSDF base, a learned blender) plug into the factory by emitting a compliant bundle. The bundle signature plus the QA reports give us provenance auditability — D-012 (truth-mode) checks signatures at handoff time.

## Alternatives considered

- **Keep training and runtime intertwined** — rejected. Continues the current debt: every Spark/Three.js work session bumps into Python toolchain issues.
- **Bundle schema as documentation only, no signing** — rejected. Without the signature, D-012's truth-mode assertion has nothing to anchor to.
- **Per-trainer bundles with no shared schema** — rejected. Defeats the boundary; runtime ends up trainer-aware again.
