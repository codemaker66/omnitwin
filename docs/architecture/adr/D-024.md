# D-024 - Scene Authority Map: Reconstruction Strategy and Transform Artifacts

Status: accepted
Date: 2026-04-30
Authors: Blake
Supersedes: none
Depends on: D-009, D-011, D-012, D-014, D-019

## Context

D-009 defines the typed spatial-layer graph. D-011 requires regional confidence budgets. D-012 separates truth-mode geometry from generated or presentation-only geometry. D-014 defines the artifact factory boundary. D-019 commits the concrete VSIR-0 schema posture and the header-in-DB, body-in-file payload split.

Two additional reconstruction research reports examined how to produce clean mesh assets from Matterport Pro3 E57 captures for Trades Hall. The reports converge on an important correction: a venue runtime cannot treat "mesh" as a single global technique. Different regions require different reconstruction strategies, and the runtime must preserve the transform evidence that connects each asset back to the canonical venue frame.

This ADR commits the planning-level architecture only. It does not implement the reconstruction pipeline, database schema, or runtime loader.

## Decision

### Scene Authority Map owns representation routing

The Scene Authority Map is a per-region or per-fixture declaration of which representation is authoritative for each runtime concern. Venviewer does not use one universal reconstruction method for a venue. It routes representation authority by region and fixture type: broad shell geometry can come from deterministic E57 meshing, radiance can come from splats, and difficult hero fixtures can use manually aligned specialist assets when that is the most honest production answer.

Each Scene Authority Map region MUST include:

- `geometry_authority`
- `appearance_authority`
- `lighting_authority`
- `physics_authority`
- `semantic_authority`
- `interaction_authority`
- `export_authority`
- `truth_status`
- `confidence_tier`
- `provenance_refs`
- `reconstruction_strategy`
- `transform_artifact_ref`

The authority fields point to the layer or asset that should be trusted for that concern. For example, a chandelier region may use a manual artist proxy for geometry and interaction bounds, a Gaussian splat layer for appearance/radiance, semantic labels from the venue graph, and no standards export until a measured or reviewed proxy export exists. A wall plane may use an E57-derived mesh for geometry, physics, interaction, and export, with the splat layer used only for appearance.

`truth_status` distinguishes measured, inferred, generated, proxy, and presentation-enhanced regions. `confidence_tier` references D-011. `provenance_refs` reference D-012 provenance records and signed artifact-bundle entries. `transform_artifact_ref` points at the transform record that places the region's asset into the venue and/or renderer frame.

Allowed `reconstruction_strategy` values:

- `matterpak_original`
- `e57_poisson`
- `e57_bpa`
- `e57_dual_meshing`
- `realityscan_hybrid`
- `pgsr_mesh_extract`
- `twodgs_mesh_extract`
- `neural_surface_reconstruction`
- `manual_artist_proxy`
- `geometry_nodes_parametric`
- `kitbash_proxy`
- `procedural_runtime`

These values describe how the region's geometry asset was produced. They do not by themselves determine whether a region is measurement-grade. Confidence and export eligibility still come from D-011 metrics and D-012 provenance.

### Production reconstruction policy

For Trades Hall and similar venues, the default production policy is:

- **Default room shell:** deterministic E57-to-mesh path using PDAL, CloudCompare, Open3D, and Blender, emitted as GLB. MatterPak OBJ remains an allowed reference or fallback source, not the default production authority when E57 is available.
- **Ornate plaster, dome, and frieze:** deterministic geometry first. RealityScan, PGSR mesh extraction, and 2DGS mesh extraction may be compared as alternatives, but they do not replace the deterministic baseline without metrics and visual review.
- **Chandeliers, crystal, stained glass, mirrors, and polished reflective regions:** manual or artist proxy is expected, not a failure. These regions are difficult reconstruction subjects; forcing them through a general mesh pipeline will produce false precision.
- **Gaussian splat layers:** remain appearance/radiance/lighting authority where appropriate. A splat can be the best visual truth for a region even when the operational geometry comes from a separate deterministic mesh or proxy.

### Hero Region Specialists

Hero Region Specialists are the explicit pattern for regions where generic reconstruction is expected to fail or produce misleading geometry. The region classifier MUST support at least:

- `chandelier`
- `crystal`
- `stained_glass`
- `mirror`
- `polished_metal`
- `ornate_plasterwork`
- `dome_frieze`
- `carved_wood`
- `marble_fireplace`

Generic reconstruction is not allowed to decide these regions alone. Specialist handling may use manual artist proxy, deterministic parametric geometry, kitbash proxy, procedural runtime geometry, or a measured mesh if the data supports it. The Scene Authority Map records that choice. Truth Mode must be able to show whether the region is measured, inferred, generated, proxy, splat-authoritative, or presentation-enhanced.

Expected production and fallback policy:

| Region class | Expected production strategy | Fallback strategy |
|---|---|---|
| `chandelier` | Manual or artist-assisted proxy aligned to E57 scale and fixture anchor points; splat remains appearance/radiance authority. | Simplified kitbash/procedural proxy with explicit proxy truth status and no measurement-grade export claim. |
| `crystal` / crystal pendants | Artist proxy or clustered procedural geometry for silhouette and interaction clearance; splat for sparkle/radiance. | Appearance-only splat authority with simplified collision/clearance bounds. |
| `stained_glass` | Plane or shallow-relief measured/proxy frame plus authored translucent material; E57 or panorama references for scale and placement. | Splat or panorama-derived appearance panel marked presentation-enhanced. |
| `mirror` | Proxy plane with reflective material and explicit non-measured reflection behavior; measured frame geometry where available. | Opaque/low-reflective placeholder plane marked presentation-enhanced to avoid false reflected geometry. |
| `polished_metal` | Proxy or deterministic base geometry with authored material; splat for radiance where reflection detail matters. | Simplified material/geometry proxy with confidence downgrade. |
| `ornate_plasterwork` | Deterministic E57 mesh first, with RealityScan, PGSR, 2DGS, or neural-surface comparison allowed against metrics. | Geometry Nodes parametric or artist proxy for repeated motifs, labeled inferred or proxy. |
| `dome_frieze` | Deterministic geometric scaffold from E57 plus parametric/artist detail pass; compare neural extraction only after baseline. | Simplified dome/frieze proxy with splat appearance authority. |
| `carved_wood` | E57 or MatterPak base mesh where clean enough, augmented by proxy/kitbash detail for carvings. | Artist proxy or simplified measured bounding geometry plus splat appearance. |
| `marble_fireplace` | Deterministic mesh or MatterPak base for massing, artist material/detail pass for veining and carved edges. | Proxy mesh aligned to E57 with splat/panorama appearance authority. |

### Coordinate convention

VSIR and runtime manifests MUST use these frame names consistently:

- **CVF:** Canonical Venue Frame. Metres, Matterport/E57-derived, Z-up unless explicitly documented otherwise. This is the authoritative venue-local metric frame.
- **ARF:** Asset Runtime Frame. The local frame in which an emitted asset file is authored.
- **RRF:** Render Runtime Frame. The frame expected by the renderer, including any Three.js/R3F orientation and scaling conventions.

Every nontrivial asset MUST persist an `ARF -> CVF` transform, a `CVF -> RRF` transform, or both, depending on where the asset enters the runtime. Identity transforms may be omitted only for trivial assets whose frame is explicitly declared identical to the target frame.

### Runtime manifests require TransformArtifactV0

Runtime manifests must support a `TransformArtifactV0` record for transforms that are load-bearing or nontrivial.

Required fields:

- `sourceFrame`
- `targetFrame`
- `units`
- `matrix` - 4x4 column-major Matrix4d
- `alignmentMethod`
- `residualRmseM`
- `landmarks`
- `provenance`
- `creator`
- `reviewer`
- `date`

`alignmentMethod` records how the transform was obtained, such as manual alignment, ICP, landmark solve, Matterport/E57 extraction, or Blender-authored placement. `residualRmseM` is nullable only when no measured residual exists; absence of a residual is itself a confidence signal. `landmarks` records optional named or typed source/target correspondence pairs used in the solve. `provenance` follows D-012 semantics. `creator`, `reviewer`, and `date` record human accountability for manual or artist-assisted transforms.

Transform artifacts are signed bundle contents, not public copy. They exist so that a future reviewer can answer, "what coordinate space is this asset in, and how do we know?"

## Consequences

The reconstruction pipeline can advance region by region without pretending one method solves the entire venue. Room shells can be deterministic and metric. Ornate or reflective hero regions can use specialist assets without being mislabeled as measured mesh. Splat rendering remains a radiance/appearance authority, not an all-purpose geometry authority.

T-091 planning now has a concrete next split:

- define `TransformArtifactV0` in the runtime manifest surface,
- spike deterministic E57 meshing for the room shell,
- spike hero-region asset production for difficult regions,
- compare RealityScan/PGSR/2DGS only after the deterministic baseline exists.

No PostgreSQL schema, VSIR table, or full reconstruction pipeline is implemented by this ADR.

## Alternatives considered

- **Single global reconstruction strategy per venue.** Rejected. Trades Hall has walls, dome/frieze, chandeliers, reflective/crystal regions, carved timber, and marble. One strategy would either underfit difficult regions or overcomplicate simple room shell geometry.
- **Treat manual proxies as failure.** Rejected. For chandeliers, crystal, stained glass, mirrors, and polished metal, a proxy can be the most honest production asset when labeled correctly.
- **Let each asset carry ad hoc transform notes.** Rejected. Coordinate ambiguity is a core failure mode for mixed E57, Matterport, Blender, and renderer outputs. A typed transform artifact is required.
- **Make Gaussian splats the only authority.** Rejected. Splats are excellent appearance/radiance assets but are not sufficient by themselves for all operational geometry, collision, measurement, or interchange needs.
