# D-002 — Three.js ≥ 0.180 required

Status: Accepted
Date: 2026-04-23
Authors: Blake
Supersedes: none

## Context

Spark 2.0 (D-001) depends on Three.js 0.180+ APIs for SplatMesh integration and the updated renderer pipeline. R3F 8.17 is compatible with Three 0.170; R3F 9.x supports Three 0.180+. Delaying the upgrade makes the upgrade harder every week as other deps drift forward. The codebase is currently on 0.170.0; the upgrade is required before any Spark integration work begins.

## Decision

Venviewer's minimum Three.js version is 0.180.0. The upgrade must land before any splat-rendering or hybrid-scene task.

## Consequences

- Upgrade is a pre-requisite for D-004 (projective texturing) and D-007 (three camera modes) implementation work.
- Upgrade likely forces R3F 8 → 9 migration in parallel.
- Pre-Spark tasks can proceed on 0.170; any splat-rendering or hybrid-scene task must wait for the upgrade to land.

## Alternatives considered

The original ADR did not enumerate alternatives in detail. The implicit alternative — staying on Three.js 0.170 indefinitely — is rejected because Spark 2.0 cannot run on 0.170 and the cost of the upgrade grows over time as other deps drift forward.
