# D-001 — Spark, not drei `<Splat />`

Status: Accepted
Date: 2026-04-23
Authors: Blake
Supersedes: none

## Context

Gaussian splat rendering needs a renderer that integrates cleanly with our Three.js scene graph and our chosen production splat format. Spark 2.0 (`@sparkjsdev/spark`) provides this. Spark's `SplatMesh` extends `THREE.Object3D`, enabling hybrid splat-and-mesh rendering natively in the scene graph. Spark handles spherical-harmonic-correct object-space transforms. Spark supports SPZ format, which is our production splat format. The drei alternative — `<Splat />` from `@react-three/drei` — does not have these properties; it is a prototyping tool with no LOD, no compositing, and no SH-aware transforms, and it assumes `.ply`. The previous-generation library GaussianSplats3D (mkkellogg) has been deprecated by its own author in favor of Spark.

## Decision

Production Gaussian splat renderer is Spark 2.0 (`@sparkjsdev/spark`). drei's `<Splat />` from `@react-three/drei` is banned.

## Consequences

- Three.js ≥ 0.180 required (D-002).
- `.claude/gotchas/spark-vs-drei-splat.md` enforces this at review time.
- Any splat-rendering PR is instantly reviewable: did the author import `SplatMesh` from `@sparkjsdev/spark` or `Splat` from `@react-three/drei`? The latter is a block.

## Alternatives considered

- **drei's `<Splat />` (`@react-three/drei`)** — rejected. Lacks spherical-harmonic-correct object-space transforms; assumes `.ply` only; prototyping tool with no LOD, no compositing, no SH-aware transforms.
- **GaussianSplats3D (mkkellogg)** — rejected. Deprecated by its own author in favor of Spark.
