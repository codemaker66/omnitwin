# D-070 — Files in git as sole source of truth

Status: proposed
Date: 2026-04-25
Authors: Blake
Supersedes: none

## Context

Solo founder + Claude Code as agent + multiple LLM sessions producing architecture work. Adding SaaS task management or external knowledge bases (Notion, Linear, Obsidian) creates parallel sources of truth that drift. Files in git that Claude Code reads natively solve the synchronization problem at zero infrastructure cost.

## Decision

Tasks live in `docs/state/tasks.md`. Decisions live in `docs/architecture/adr/`. State snapshots live in `docs/state/PROJECT_STATE.md`. Session logs live in `docs/sessions/YYYY-MM-DD.md`. Operational state lives in `state/*.json` and `state/*.jsonl`. No SaaS tool sits in the critical path of architectural work.

## Consequences

Every architectural artifact is in version control: diffable, branchable, mergeable. Future Claude Code sessions read the same files we do. Trade-off: requires discipline to keep files current; no tool-enforced workflow.

## Alternatives considered

- **task-master-ai** — deferred per D-072.
- **Notion** — rejected. Creates a parallel source of truth that drifts from the repo.
- **Linear** — rejected. Adds cost and a parallel truth surface.
- **GitHub Issues for architecture** — rejected. Issues are for surface bugs, not durable architecture decisions.
