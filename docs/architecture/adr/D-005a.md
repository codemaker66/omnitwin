# D-005a — View-dependent residual layer

Status: proposed
Date: 2026-04-25
Authors: Blake
Supersedes: D-005

## Context

D-005 commits to "splat cropped to reflective surfaces only" as the residual layer. Deep research synthesis confirmed the residual *role* is correct (separate view-dependent appearance from photometric base) but flagged that naming the *implementation* (3DGS splats) freezes a 2026-state implementation choice. Recent papers (2DGS, Ref-Gaussian, 3DGS-DR, Ref-GS, GaussianShader, DN-Splatter) demonstrate that better residual implementations exist for specific material classes (specular floors, chandelier crystal, plaster detail). Manual SuperSplat cropping is also flagged as the most dangerous artisanal step in the current pipeline.

## Decision

Reframe the residual layer as "view-dependent residual layer" with a `ResidualAppearance` interface. Cropped Gaussian splat remains the default v1 implementation. Future implementations (2DGS, Ref-GS, mesh-bound Gaussians via SuGaR, baked neural residual) plug in. Manual cropping is replaced by automatic residual extraction (render projective base, render full splat, compute persistent error mask, intersect with reflective/transparent semantic masks).

## Consequences

Specific material-aware residual implementations can be introduced for specific regions without disturbing the base layer. Manual SuperSplat cropping moves to v2 deprecation — the current artisanal failure mode (a missed reflective surface = visible bug) gets replaced with a deterministic, regenerable extraction pass.

## Alternatives considered

- **Keeping D-005 unchanged** — rejected. Locks splat as forever-form residual.
- **Removing the residual layer entirely** — rejected. Returns to single-renderer mesh+texture; loses chandelier and reflective-floor fidelity.
