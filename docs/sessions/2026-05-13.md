## T-419 — Smooth non-magnetic furniture drag

Completed the active 3D furniture-drag feel fix after live feedback that moving
tables near walls and other furniture felt magnetic and clunky.

Root cause:

- the active pointer-move loop in `SelectionSystem` hard-snapped the dragged
  primary item to platform edges, wall edges, and furniture alignment positions
  on every frame
- that snap happened while the mouse was still held, so the object could jump
  away from the user's grabbed point near walls or nearby tables/chairs

Changes:

- extracted `computeFluidFurnitureDragFrame` in
  `packages/web/src/lib/furniture-drag.ts`
- active dragging now computes movement directly from the floor hit and original
  grab offset
- smart alignment remains visible as guide lines only during active drag
- grouped table/chair movement, soft constraints, and save semantics are
  preserved

Verification:

- `pnpm --filter @omnitwin/web test -- furniture-drag snap-guide placement-store`
- `pnpm --filter @omnitwin/web lint`
- `pnpm --filter @omnitwin/web typecheck`
- `pnpm --filter @omnitwin/web e2e -- e2e/editor.spec.ts -g "page loads with a visible WebGL canvas|canvas accepts left and right clicks without crashing" --workers=1`
- `pnpm --filter @omnitwin/web build`

## T-420 — Camera keyboard pan input safety

Completed the camera-keyboard guard after live feedback that pressing movement
keys such as `A`/`D` while a planner menu, Draw tool, or selected furniture was
active could pan the camera far away from the hall.

Root cause:

- `CameraRig` only locked WASD/arrow panning when the key event target was a
  text/dialog element
- many planner UI states leave focus on a button or the document body, so the
  global camera pan handler still accepted movement keys while the user was
  interacting with menus/tools/objects

Changes:

- focused buttons/toolbars/menus now count as keyboard-locking UI targets
- camera keyboard pan now also suspends from planner state: catalogue drawer,
  active placement/drag, selected furniture, marquee selection, camera-reference
  draft, Laser Diagram, measurement, and guideline tools
- per-frame camera panning clears any held pan keys as soon as those UI states
  become active

Verification:

- `pnpm --filter @omnitwin/web lint`

- `pnpm --filter @omnitwin/web build`
